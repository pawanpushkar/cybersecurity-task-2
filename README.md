# 🔐 Cybersecurity Task 2 – SOC Investigation

## 📌 Overview
This project is part of my cybersecurity learning journey, focusing on Security Operations Center (SOC) investigation techniques. The main objective of this task is to analyze logs, detect suspicious activities, and understand real-world cyber threats.

---

## 🎯 Objectives
- Analyze system and network logs
- Identify suspicious login attempts
- Detect potential security threats
- Understand SOC investigation workflow
- Gain hands-on experience with cybersecurity tools

---

## 🛠️ Tools & Technologies Used
- Kali Linux
- Splunk / Log Analysis Tools
- Nmap (for scanning)
- Wireshark (optional)
- Linux Commands

---

## 🔍 Task Description
In this task, I performed log analysis to identify unusual activities such as:
- Multiple failed login attempts
- Unauthorized access patterns
- Suspicious IP behavior

The logs were carefully examined to detect potential brute-force attacks and other security threats.

---

## 🚀 Steps Performed
1. Collected log data from the system/network
2. Imported logs into analysis tool (Splunk)
3. Filtered and searched for failed login attempts
4. Identified suspicious IP addresses
5. Analyzed attack patterns
6. Documented findings

---

## 📊 Key Findings
- Detected multiple failed login attempts from unknown IP addresses
- Identified possible brute-force attack behavior
- Observed unusual login timing patterns
